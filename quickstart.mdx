## Prerequisites

Before you begin, make sure you have:

<CardGroup cols={2}>
  <Card title="Python 3.10+" icon="python">
    Python 3.10, 3.11, or 3.12 installed on your system
  </Card>
  <Card title="Google API Key" icon="key">
    Free API key from [Google AI Studio](https://aistudio.google.com)
  </Card>
</CardGroup>

<Note>
The Google API key is free and provides generous quotas. OpenRouter is optional and only needed for the `hyper` and `gemini3` presets.
</Note>

## Installation

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/timepoint-ai/timepoint-flash.git
    cd timepoint-flash
    ```
  </Step>

  <Step title="Run Setup Script">
    The setup script checks prerequisites, installs dependencies, and creates your `.env` file:

    ```bash
    ./setup.sh
    ```

    **What it does:**
    - Verifies Python 3.10+ is installed
    - Checks for pip
    - Installs all dependencies via `pip install -e .`
    - Creates `.env` from `.env.example`
    - Makes scripts executable

    <Note>
    If you prefer manual setup: `pip install -e .` then `cp .env.example .env`
    </Note>
  </Step>

  <Step title="Add Your API Key">
    Edit the `.env` file and add your Google API key:

    ```bash .env
    # === Required (at least one LLM provider) ===
    GOOGLE_API_KEY=your-key-here
    OPENROUTER_API_KEY=  # Optional

    # === Database ===
    DATABASE_URL=sqlite+aiosqlite:///./timepoint.db

    # === Auth (optional - for iOS app mode) ===
    AUTH_ENABLED=false
    ```

    <Warning>
    Don't commit your API keys to version control. The `.env` file is already in `.gitignore`.
    </Warning>
  </Step>
</Steps>

## Manual Server Start

If you prefer to start the server manually:

<CodeGroup>
```bash Development (with auto-reload)
./run.sh -r
# Starts on http://localhost:8000
```

```bash Production
./run.sh
```


- **Interactive docs**: `http://localhost:8000/docs`
- **API v1**: `http://localhost:8000/api/v1/`
</Note>

## Your First API Call

Once the server is running, generate your first scene:

<CodeGroup>
```bash cURL (Streaming)
curl -X POST http://localhost:8000/api/v1/timepoints/generate/stream \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Alan Turing breaks Enigma at Bletchley Park Hut 8, winter 1941",
    "preset": "balanced",
    "generate_image": true
  }'
```

```python Python
import httpx

response = httpx.post(
    "http://localhost:8000/api/v1/timepoints/generate/sync",
    json={
        "query": "Alan Turing breaks Enigma at Bletchley Park Hut 8, winter 1941",
        "preset": "balanced",
        "generate_image": True
    },
    timeout=180.0
)

timepoint = response.json()
print(f"Generated: {timepoint['location']}, {timepoint['year']}")
```

```javascript JavaScript
const response = await fetch('http://localhost:8000/api/v1/timepoints/generate/sync', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    query: 'Alan Turing breaks Enigma at Bletchley Park Hut 8, winter 1941',
    preset: 'balanced',
    generate_image: true
  })
});

const timepoint = await response.json();
console.log(`Generated: ${timepoint.location}, ${timepoint.year}`);
```
</CodeGroup>

### Generation Endpoints

Flash provides three generation modes:

<CardGroup cols={3}>
  <Card title="Streaming (SSE)" icon="tower-broadcast">
    **Recommended** for real-time progress updates
    
    `POST /api/v1/timepoints/generate/stream`
    
    Returns Server-Sent Events as each agent completes
  </Card>
  
  <Card title="Synchronous" icon="clock">
    **Simple** blocking request
    
    `POST /api/v1/timepoints/generate/sync`
    
    Waits until complete, returns full result
  </Card>
  
  <Card title="Background" icon="list-check">
    **Async** for long-running tasks
    
    `POST /api/v1/timepoints/generate`
    
    Returns task ID, poll for result
  </Card>
</CardGroup>

## Quality Presets

Choose a preset based on your speed vs. quality needs:

<CodeGroup>
```bash Fast (hyper)
# ~55 seconds, requires OPENROUTER_API_KEY
curl -X POST localhost:8000/api/v1/timepoints/generate/sync \
  -d '{"query": "...", "preset": "hyper"}'
```

```bash Balanced (default)
# ~90-110 seconds, best for production
curl -X POST localhost:8000/api/v1/timepoints/generate/sync \
  -d '{"query": "...", "preset": "balanced"}'
```

```bash High Quality (hd)
# ~2-2.5 minutes, extended thinking models
curl -X POST localhost:8000/api/v1/timepoints/generate/sync \
  -d '{"query": "...", "preset": "hd"}'
```

```bash Latest Model (gemini3)
# ~60 seconds, requires OPENROUTER_API_KEY
curl -X POST localhost:8000/api/v1/timepoints/generate/sync \
  -d '{"query": "...", "preset": "gemini3"}'
```
</CodeGroup>

## Interactive Features

Once you've generated a scene, you can interact with it:

<Steps>
  <Step title="Chat with Characters">
    ```bash
    curl -X POST localhost:8000/api/v1/interactions/{timepoint_id}/chat \
      -d '{
        "character_name": "Alan Turing",
        "message": "How did you figure out the Enigma settings?"
      }'
    ```
    
    Characters respond in-character with period-appropriate knowledge.
  </Step>
  
  <Step title="Time Travel">
    ```bash
    # Jump forward 1 hour
    curl -X POST localhost:8000/api/v1/temporal/{timepoint_id}/next \
      -d '{"units": 1, "unit": "hour"}'
    
    # Jump back 1 day
    curl -X POST localhost:8000/api/v1/temporal/{timepoint_id}/prior \
      -d '{"units": 1, "unit": "day"}'
    ```
  </Step>
  
  <Step title="Survey Characters">
    ```bash
    curl -X POST localhost:8000/api/v1/interactions/{timepoint_id}/survey \
      -d '{"question": "What do you think will happen next?"}'
    ```
    
    Get responses from all characters at once.
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Server won't start">
    **Check your .env file:**
    ```bash
    # Make sure GOOGLE_API_KEY is set (not placeholder)
    cat .env | grep GOOGLE_API_KEY
    ```
    
    **Check database:**
    ```bash
    # Run migrations
    alembic upgrade head
    ```
    
    **Check port availability:**
    ```bash
    # Port 8000 might be in use
    lsof -i :8000
    # Use a different port
    uvicorn app.main:app --port 8080
    ```
  </Accordion>
  
  <Accordion title="Generation fails with API error">
    **Verify API key works:**
    ```bash
    curl -H "Authorization: Bearer $GOOGLE_API_KEY" \
      https://generativelanguage.googleapis.com/v1/models
    ```
    
    **Check quotas:**
    Visit [Google AI Studio](https://aistudio.google.com) and check your quota limits.
    
    **Try a different preset:**
    ```bash
    # If 'balanced' fails, try 'hyper' (requires OpenRouter)
    curl -X POST localhost:8000/api/v1/timepoints/generate/sync \
      -d '{"query": "...", "preset": "hyper"}'
    ```
  </Accordion>
  
  <Accordion title="Images not generating">
    **Check image providers:**
    ```bash
    curl localhost:8000/api/v1/models/providers
    ```
    
    Flash uses 3-tier fallback, so images should always work. If all providers fail:
    - Google API key might not have Imagen access
    - Network connectivity issues
    - Set `generate_image: false` to skip images
  </Accordion>
  
  <Accordion title="Slow generation">
    **Try a faster preset:**
    ```bash
    # hyper is ~55 seconds vs balanced ~110 seconds
    {"preset": "hyper"}
    ```
    
    **Disable image generation:**
    ```bash
    {"generate_image": false}  # Saves 10-20 seconds
    ```
    
    **Check rate limits:**
    If you're hitting rate limits, generation will slow down with retries.
  </Accordion>
</AccordionGroup>

## Testing Your Setup

Run the test suite to verify everything works:

```bash
# Fast unit tests (no API calls)
python3.10 -m pytest tests/ -m fast -v

# Integration tests (with mocked APIs)
python3.10 -m pytest tests/ -m integration -v

# All tests (522 fast + integration, 11 skipped e2e)
python3.10 -m pytest tests/ -v
```

<Note>
End-to-end tests are skipped by default because they require real API keys and make actual API calls. To run them: `pytest tests/ -m e2e -v`
</Note>
