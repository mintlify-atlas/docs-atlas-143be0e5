TIMEPOINT Flash uses a pluggable billing system via the `BillingProvider` protocol (see `app/services/billing.py:1`).

## Architecture

The open-source app ships with `NoOpBilling` (unlimited access). You can swap in a custom billing provider for credit management.

```
App → BillingProvider Protocol → Your Implementation
                               ↓
                         check_credits()
                         on_credits_granted()
```

## BillingProvider Protocol

Defined in `app/services/billing.py:12`:

```python
from typing import Protocol

class BillingProvider(Protocol):
    """Abstract billing provider. Private modules override this."""

    async def check_credits(self, user_id: str, cost: int) -> bool:
        """Check if user has enough credits.

        Args:
            user_id: User identifier (JWT sub claim)
            cost: Cost in credits

        Returns:
            True if user has enough credits, False otherwise
        """
        ...

    async def on_credits_granted(self, user_id: str, amount: int, source: str) -> None:
        """Notify when credits are granted.

        Args:
            user_id: User identifier
            amount: Credits granted
            source: Source of credits (e.g., "signup", "admin", "purchase")
        """
        ...
```

## Default: NoOpBilling

The default implementation allows all operations:

```python
class NoOpBilling:
    """Default: open access, no billing.

    Used when timepoint-billing is not installed.
    """

    async def check_credits(self, user_id: str, cost: int) -> bool:
        return True  # Always allow

    async def on_credits_granted(self, user_id: str, amount: int, source: str) -> None:
        pass  # No-op
```

    import logging
    from typing import Protocol

    logger = logging.getLogger(__name__)


    class CustomBillingProvider:
        """Billing provider with PostgreSQL credit tracking."""

        def __init__(self, db_url: str):
            self.db_url = db_url

        async def check_credits(self, user_id: str, cost: int) -> bool:
            """Check if user has enough credits.

            Args:
                user_id: User identifier from JWT
                cost: Cost in credits (e.g., 10 for scene generation)

            Returns:
                True if sufficient credits, False otherwise
            """
            # Query your database
            user = await self._get_user(user_id)
            if user is None:
                logger.warning(f"User {user_id} not found")
                return False

            has_credits = user.credits >= cost

                await self._deduct_credits(user_id, cost)

                logger.warning(f"User {user_id}: insufficient credits ({user.credits} < {cost})")

            """Notify when credits are granted.

## Credit Costs

Default credit costs per operation:

| Operation | Cost | Endpoint |
|-----------|------|----------|
| Scene generation | 10 | `POST /api/v1/generate` |
| Temporal jump | 5 | `POST /api/v1/temporal/{id}/next` |
| Character chat | 2 | `POST /api/v1/timepoints/{id}/chat` |
| Dialog extension | 3 | `POST /api/v1/timepoints/{id}/extend-dialog` |
| Survey | 5 | `POST /api/v1/timepoints/{id}/survey` |

Costs are checked before execution. If `check_credits()` returns `False`, the request fails with 402 Payment Required.

## Credit Granting

Credits are granted via:

### 1. Signup Credits

New users receive free credits (see `app/config.py:388`):

```bash .env
SIGNUP_CREDITS=50  # Free credits on first sign-in
```

When a user signs in for the first time:

```python
await billing_provider.on_credits_granted(
    user_id=user.id,
    amount=settings.SIGNUP_CREDITS,
    source="signup"
)
```

### 2. Admin Grant

Admins can grant credits via API:

```bash
curl -X POST http://localhost:8000/api/v1/admin/credits/grant \
  -H "X-Admin-Key: your-admin-key" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user_123",
    "amount": 100,
    "reason": "promotional credits"
  }'
```

### 3. Purchase (Your Implementation)

Integrate with payment processor:

```python
# After successful Stripe payment
await billing_provider.on_credits_granted(
    user_id=user_id,
    amount=credits_purchased,
    source="purchase"
)
```

## Database Schema Example

Example schema for credit tracking:

```sql
CREATE TABLE user_credits (
    user_id VARCHAR(255) PRIMARY KEY,
    credits INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE credit_transactions (
    id SERIAL PRIMARY KEY,
    user_id VARCHAR(255) NOT NULL,
    amount INTEGER NOT NULL,  -- Positive for grant, negative for deduction
    source VARCHAR(100) NOT NULL,  -- 'signup', 'admin', 'purchase', 'generation', etc.
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user_credits(user_id)
);

CREATE INDEX idx_user_transactions ON credit_transactions(user_id, created_at);
```

## Redis Caching (Optional)

Cache credit balances for performance:

```python
import redis.asyncio as redis

class CachedBillingProvider:
    def __init__(self, db_url: str, redis_url: str):
        self.db_url = db_url
        self.redis = redis.from_url(redis_url)

    async def check_credits(self, user_id: str, cost: int) -> bool:
        # Check cache first
        cached = await self.redis.get(f"credits:{user_id}")
        if cached:
            credits = int(cached)
        else:
            # Fetch from database
            user = await self._get_user(user_id)
            credits = user.credits
            # Cache for 60 seconds
            await self.redis.set(f"credits:{user_id}", credits, ex=60)

        if credits >= cost:
            # Deduct and invalidate cache
            await self._deduct_credits(user_id, cost)
            await self.redis.delete(f"credits:{user_id}")
            return True
        return False
```

## Microservice Pattern

For larger deployments, use a separate billing service:

```python
import httpx

class BillingServiceProvider:
    """Billing provider that proxies to a microservice."""

    def __init__(self, service_url: str, api_key: str):
        self.service_url = service_url
        self.api_key = api_key
        self.client = httpx.AsyncClient()

    async def check_credits(self, user_id: str, cost: int) -> bool:
        response = await self.client.post(
            f"{self.service_url}/check",
            json={"user_id": user_id, "cost": cost},
            headers={"X-API-Key": self.api_key},
        )
        return response.json()["has_credits"]

    async def on_credits_granted(self, user_id: str, amount: int, source: str) -> None:
        await self.client.post(
            f"{self.service_url}/grant",
            json={"user_id": user_id, "amount": amount, "source": source},
            headers={"X-API-Key": self.api_key},
        )
```

## Error Handling

Handle billing failures gracefully:

```python
try:
    has_credits = await billing_provider.check_credits(user.id, cost=10)
except Exception as e:
    logger.error(f"Billing check failed: {e}")
    # Fail open or closed depending on your policy
    raise HTTPException(status_code=503, detail="Billing service unavailable")
```

<Warning>
Decide on a fail-open or fail-closed policy:
- **Fail-open**: Allow requests if billing service is down (better UX, risk of free usage)
- **Fail-closed**: Reject requests if billing service is down (safer, worse UX)
</Warning>

<Tip>
Use the `NoOpBilling` provider during development and switch to your custom provider in production.
</Tip>
