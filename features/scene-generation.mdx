TIMEPOINT Flash generates complete historical scenes using a 14-agent pipeline with parallel execution, Google Search grounding, and automatic quality control.

## How It Works

The generation pipeline runs in three phases:

<Steps>
  <Step title="Foundation (Sequential)">
    **Judge** → **Grounding** → **Timeline** → **Scene**
    
    - Validates your query
    - Verifies facts via Google Search (for historical events)
    - Extracts temporal coordinates
    - Generates setting and atmosphere
  </Step>
  
  <Step title="Parallel Generation">
    **Characters** + **Moment** + **Camera** run simultaneously
    
    - Identifies up to 6 characters with roles
    - Generates relationship graph
    - Creates character bios with voice differentiation
    - Defines dramatic tension arc
    - Sets camera composition
  </Step>
  
  <Step title="Dialog & Image (Sequential)">
    **Dialog** → **Critique** → **ImagePrompt** → **Optimizer** → **ImageGen**
    
    - Generates up to 7 period-appropriate dialog lines
    - Reviews for anachronisms and cultural errors (auto-retry if critical)
    - Assembles detailed image prompt
    - Compresses and physicalizes emotion (~77 words)
    - Generates photorealistic image (3-tier fallback)
  </Step>
</Steps>

## Example: AlphaGo Move 37

**Query:**
```
AlphaGo plays Move 37 against Lee Sedol in Game 2, 
the move no human would make, Four Seasons Hotel Seoul March 10 2016
```

**Generated Scene:**

```
Location:    Four Seasons Hotel, Seoul, South Korea
Date:        March 10, 2016 — afternoon, spring
Tension:     high

Setting:     A brightly lit, modern conference room arranged for a Go tournament.
             Central Go board on a low, polished wooden table. Lee Sedol sits
             opposite the AlphaGo system, represented by a monitor displaying its
             moves. Journalists, AI researchers, and Go enthusiasts seated in rows
             behind a low barrier. Cables snake across the floor.

Atmosphere:  Intense concentration and quiet anticipation. The audience is hushed.
             A palpable sense of history being made, mixed with disbelief at
             AlphaGo's unconventional moves.
```

**Characters:**

| Name | Role | Voice |
|------|------|-------|
| Lee Sedol | primary | Short, halting fragments — stunned understatement |
| AlphaGo | primary, silent | Represented by monitor (non-human entity) |
| Commentator 1 | secondary | Chatty, analytical, comfortable on-air cadence |
| Commentator 2 | secondary | More measured, technical observations |
| Tournament Official | secondary | Formal, procedural |

**Dialog:**

> **Lee Sedol:** "...Huh. That's... certainly a move."
>
> **Commentator 1:** "Well, that's one way to put it. I'm not sure I've ever seen a professional play quite like that before."
>
> **Lee Sedol:** "It's... unexpected, to say the least. Maybe it's a bluff?"
>
> **Commentator 1:** "A bluff? Against AlphaGo? Now that's a thought."
>
> **Commentator 2:** "It's a bold one, if so. I'm not sure even a human could read that."
>
> **Commentator 1:** "It's either genius or madness, and I honestly can't tell which right now."
>
> **Lee Sedol:** "Perhaps both, a little? It's making things... interesting."

## Key Features

### Google Search Grounding

Historical events and figures are verified via Google Search before generation:

- **Verified locations:** "Equitable Center, 35th floor, Manhattan" (not "an IBM room")
- **Verified dates:** Exact match to historical records
- **Physical presence:** Non-human entities (Deep Blue, AlphaGo) correctly represented by their operators/monitors
- **Event mechanics:** How the event physically worked (board setup, room layout, visible technology)

See [Grounding](/concepts/grounding) for technical details.

### Voice Differentiation

Each character gets a **social register** that constrains:

- Sentence structure (elite vs. common)
- Vocabulary (Latinate vs. Germanic roots)
- Verbal tics and speech patterns

Characters must be **identifiable by voice alone** — no "everyone sounds the same" dialog.

<Note>
The dialog agent uses **sequential roleplay**: each line is generated separately with the character's bio as the system prompt, creating authentic personality-driven dialog.
</Note>

### Narrative Arc System

Every scene follows a narrative shape (Vonnegut + Freytag):

| Shape | Pattern | Example |
|-------|---------|--------|
| Man in Hole | Good → Bad → Good | Rescue missions, comebacks |
| Creation | Low → Steady Rise | Apollo 11, Woodstock |
| Cinderella | Low → High → Low → Very High | Rags to riches |
| From Bad to Worse | Bad → Worse | Pompeii, Titanic |
| Old Testament | Rise → Deep Fall | Icarus, Sarajevo assassination |
| Freytag | Exposition → Climax → Resolution | Standard dramatic pyramid |

Dialog lines are assigned narrative functions: `establish`, `complicate`, `escalate`, `turn`, `react`, `resolve`, `punctuate`.

### Critique Loop

Dialog is automatically reviewed for:

- **Anachronisms:** Modern phrases in historical contexts
- **Cultural errors:** Greek deities in Roman settings
- **Voice problems:** Characters sounding too similar
- **Historical inaccuracies:** Wrong knowledge for the time period

If **critical issues** are found, dialog regenerates automatically with corrections.

## Generation API

### Streaming Generation (Recommended)

```bash
curl -X POST http://localhost:8000/api/v1/timepoints/generate/stream \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Trinity test 5:29 AM July 16 1945",
    "generate_image": true,
    "preset": "balanced"
  }'
```

**Server-Sent Events (SSE):**

```
data: {"event": "step", "step": "judge", "status": "success"}
data: {"event": "step", "step": "grounding", "status": "success", "data": {"verified_location": "..."}}
data: {"event": "step", "step": "timeline", "status": "success"}
data: {"event": "step", "step": "scene", "status": "success"}
data: {"event": "step", "step": "characters", "status": "success"}
data: {"event": "step", "step": "dialog", "status": "success"}
data: {"event": "step", "step": "image_generation", "status": "success"}
data: {"event": "complete", "timepoint_id": "abc123", "slug": "trinity-test-1945-xyz"}
```

<Tip>
Use streaming to show progress to users. Each step completes in 1-5 seconds.
</Tip>

### Synchronous Generation

```bash
curl -X POST http://localhost:8000/api/v1/timepoints/generate/sync \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Rome 50 BCE",
    "generate_image": false,
    "preset": "hyper"
  }'
```

Blocks until complete, returns full timepoint object.

### Background Generation

```bash
# Start generation
curl -X POST http://localhost:8000/api/v1/timepoints/generate \
  -d '{"query": "Gavrilo Princip Sarajevo 1914"}'

# Response: {"timepoint_id": "abc123", "status": "pending"}

# Poll for result
curl http://localhost:8000/api/v1/timepoints/abc123
```
