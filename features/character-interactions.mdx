Interact with characters from generated scenes through three modes: one-on-one chat, dialog extension, and multi-character surveys.

<Note>
**Visibility & Auth:** All interaction endpoints check timepoint visibility. Private timepoints return 403 for non-owners. When `AUTH_ENABLED=true`, interactions require Bearer JWT and deduct credits.
</Note>

## Character Chat

Have conversations with individual characters using their full biographical context and voice.

### How It Works

The character chat agent uses **roleplay generation**:

1. **System prompt = character bio:** The character's full biography, personality, speaking style, and voice notes become the system prompt
2. **Context preservation:** Scene setting, year, location, and emotional state inform responses
3. **Conversation history:** Previous messages are included for multi-turn conversations
4. **Voice consistency:** Responses maintain the character's speech patterns and register

### Example: Questioning Lee Sedol

```bash
curl -X POST http://localhost:8000/api/v1/interactions/abc123/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "character": "Lee Sedol",
    "message": "What went through your mind when you saw Move 37?"
  }'
```

**Response:**

```json
{
  "character_name": "Lee Sedol",
  "response": "...It was... I've never seen anything like it. Not in twenty years of professional play. For a moment I thought perhaps the system had made an error. But no. It was... deliberate. Unconventional, yes, but there was an intelligence behind it that I... I didn't expect.",
  "session_id": "sess_xyz789",
  "emotional_tone": "contemplative",
  "latency_ms": 1200
}
```

### Streaming Chat

Get responses token-by-token via Server-Sent Events:

```bash
curl -X POST http://localhost:8000/api/v1/interactions/abc123/chat/stream \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "character": "Benjamin Franklin",
    "message": "What do you think of this document?"
  }'
```

**SSE Stream:**

```
data: {"event": "token", "data": "I ", "character_name": "Benjamin Franklin"}
data: {"event": "token", "data": "find ", "character_name": "Benjamin Franklin"}
data: {"event": "token", "data": "it ", "character_name": "Benjamin Franklin"}
data: {"event": "token", "data": "rather ", "character_name": "Benjamin Franklin"}
data: {"event": "token", "data": "well ", "character_name": "Benjamin Franklin"}
data: {"event": "token", "data": "composed", "character_name": "Benjamin Franklin"}
data: {"event": "done", "data": "<full response>", "character_name": "Benjamin Franklin"}
```

<Tip>
Use streaming for real-time chat interfaces to provide immediate feedback.
</Tip>

### Multi-Turn Conversations

Use `session_id` to maintain conversation context:

```bash
# First message
curl -X POST .../chat -d '{
  "character": "Oppenheimer",
  "message": "What did you feel at that moment?"
}'
# Response includes session_id: "sess_abc123"

# Follow-up message
curl -X POST .../chat -d '{
  "character": "Oppenheimer",
  "message": "Did you expect it to work?",
  "session_id": "sess_abc123"
}'
```

The character remembers the prior exchange and responds with context.

### Model Override

Specify a custom model for character responses:

```bash
curl -X POST .../chat -d '{
  "character": "Lee Sedol",
  "message": "What's your strategy now?",
  "model": "google/gemini-2.0-flash-001"
}'
```

Supported models:
- `gemini-2.5-flash` (default, fast)
- `google/gemini-2.0-flash-001` (Google native)
- `openrouter.ai/google/gemini-2.0-flash-exp:free` (OpenRouter)
- Any model configured in your router

### Response Formats

<CodeGroup>

```bash Auto (Default)
curl -X POST .../chat -d '{
  "character": "Franklin",
  "message": "Your thoughts?",
  "response_format": "auto"
}'
# Model decides based on capabilities
```

```bash Structured (JSON)
curl -X POST .../chat -d '{
  "character": "Franklin",
  "message": "Your thoughts?",
  "response_format": "structured"
}'
# Returns: {"response": "...", "in_character": true, "emotional_tone": "..."}
```

```bash Plain Text
curl -X POST .../chat -d '{
  "character": "Franklin",
  "message": "Your thoughts?",
  "response_format": "text"
}'
# Returns simple string response
```

</CodeGroup>

## Dialog Extension

Generate additional conversation between characters.

### How It Works

The dialog extension agent:

1. **Loads scene context:** Full scene, characters, relationships, tension
2. **Sequential generation:** Each line generated by "becoming" that character (bio as system prompt)
3. **Relationship-aware:** Uses the relationship graph to inform interactions
4. **Narrative arc:** Lines follow emotional beats (establish → complicate → resolve)

### Generate Additional Lines

```bash
curl -X POST http://localhost:8000/api/v1/interactions/abc123/dialog \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "characters": ["Benjamin Franklin", "John Adams"],
    "num_lines": 5,
    "prompt": "They begin discussing the wording of the document"
  }'
```

**Response:**

```json
{
  "dialog": [
    {"speaker": "Benjamin Franklin", "text": "The phrase 'unalienable rights' has a certain ring to it."},
    {"speaker": "John Adams", "text": "Indeed. It strikes at the heart of tyranny."},
    {"speaker": "Benjamin Franklin", "text": "Though I wonder if 'inalienable' might be clearer to the common man."},
    {"speaker": "John Adams", "text": "Clarity is admirable, but this is a document for the ages."},
    {"speaker": "Benjamin Franklin", "text": "Then let it be both clear and eternal."}
  ],
  "context": "Franklin and Adams debate constitutional language",
  "characters_involved": ["Benjamin Franklin", "John Adams"],
  "latency_ms": 3400
}
```

### All Characters

Omit `characters` or use `"all"` to include everyone:

```bash
curl -X POST .../dialog -d '{
  "characters": "all",
  "num_lines": 7,
  "prompt": "The tension in the room reaches a breaking point"
}'
```

The dialog agent automatically rotates speakers and handles up to 6 characters.

### Streaming Dialog

Get lines as they're generated:

```bash
curl -X POST http://localhost:8000/api/v1/interactions/abc123/dialog/stream \
  -H "Content-Type: application/json" \
  -d '{
    "characters": ["Oppenheimer", "Groves"],
    "num_lines": 5
  }'
```

**SSE Stream:**

```
data: {"event": "line", "data": {"speaker": "Oppenheimer", "text": "It's done."}}
data: {"event": "line", "data": {"speaker": "Groves", "text": "Was there ever any doubt?"}}
data: {"event": "line", "data": {"speaker": "Oppenheimer", "text": "...Always."}}
data: {"event": "done", "data": {"characters_involved": ["Oppenheimer", "Groves"]}}
```

## Character Survey

Ask all characters the same question(s) and get structured responses.

### How It Works

The survey agent:

1. **Parallel execution (default):** Each character responds simultaneously
2. **Individual roleplay:** Each response uses the character's bio as system prompt
3. **Sentiment analysis:** Automatic detection of positive/negative/neutral sentiment
4. **Summary generation:** Synthesizes all responses into key themes

### Survey All Characters

```bash
curl -X POST http://localhost:8000/api/v1/interactions/abc123/survey \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "questions": ["What do you fear most about this moment?"],
    "include_summary": true
  }'
```

**Response:**

```json
{
  "timepoint_id": "abc123",
  "questions": ["What do you fear most about this moment?"],
  "responses": [
    {
      "character_name": "John Adams",
      "question": "What do you fear most about this moment?",
      "response": "That we shall all hang for this. Separately.",
      "sentiment": "negative",
      "key_points": ["fear of execution", "consequences of treason"],
      "emotional_tone": "anxious"
    },
    {
      "character_name": "Benjamin Franklin",
      "question": "What do you fear most about this moment?",
      "response": "That we might fail not in the attempt, but in the execution of our principles afterward.",
      "sentiment": "contemplative",
      "key_points": ["long-term vision", "implementation concerns"],
      "emotional_tone": "thoughtful"
    },
    {
      "character_name": "Thomas Jefferson",
      "question": "What do you fear most about this moment?",
      "response": "That the words may not be strong enough to carry the weight of what we mean.",
      "sentiment": "negative",
      "key_points": ["linguistic precision", "lasting impact"],
      "emotional_tone": "concerned"
    }
  ],
  "summary": "The founders express a mixture of fear and determination. Adams worries about immediate consequences, Franklin about long-term execution of principles, and Jefferson about the document's lasting power. Despite their fears, all three show commitment to the cause.",
  "mode": "parallel",
  "total_characters": 3,
  "latency_ms": 2100
}
```

### Multiple Questions

Ask several questions at once:

```bash
curl -X POST .../survey -d '{
  "questions": [
    "What do you fear most?",
    "What gives you hope?",
    "What would you change about this moment if you could?"
  ]
}'
```

Each character answers all questions in sequence.

### Sequential Mode with Chaining

Characters can see prior responses:

```bash
curl -X POST .../survey -d '{
  "questions": ["Should we proceed?"],
  "mode": "sequential",
  "chain_prompts": true
}'
```

In sequential mode:
- Character A responds first
- Character B sees A's response and can react to it
- Character C sees both A and B's responses
- Creates a more dynamic group discussion

### Subset of Characters

```bash
curl -X POST .../survey -d '{
  "characters": ["Lee Sedol", "Commentator 1"],
  "questions": ["What just happened?"]
}'
```

### Streaming Survey

Get responses as each character finishes:

```bash
curl -X POST http://localhost:8000/api/v1/interactions/abc123/survey/stream \
  -H "Content-Type: application/json" \
  -d '{
    "questions": ["What are you thinking right now?"]
  }'
```

### List Sessions for a Timepoint


## Implementation Details

### Character Bio as System Prompt

From `app/agents/character_chat.py:206-239`:

```python
def _build_character_bio(self, character: Character) -> str:
    """Build character biography for system prompt."""
    lines = []
    
    if character.historical_note:
        lines.append(f"HISTORICAL CONTEXT: {character.historical_note}")
    
    if character.personality:
        lines.append(f"PERSONALITY: {character.personality}")
    
    if character.speaking_style:
        lines.append(f"SPEAKING STYLE: {character.speaking_style}")
    
    if character.voice_notes:
        lines.append(f"VOICE: {character.voice_notes}")
    
    if character.emotional_state:
        lines.append(f"CURRENT EMOTIONAL STATE: {character.emotional_state}")
    
    return "\n".join(lines)
```

The character's full context becomes the system prompt, ensuring responses are in-character.

### Sequential Dialog Generation

From `app/agents/dialog.py:251-276`:

Each dialog line is generated separately:

1. Character's bio → system prompt
2. Conversation history → user context
3. Relationship dynamics → additional context
4. Generate one line as that character
5. Add line to history
6. Move to next speaker

This creates authentic, personality-driven dialog where each character truly "speaks."
